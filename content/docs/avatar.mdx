<Steps>
<Step>
Install the Avatar component using the nocta-ui CLI:

```bash lineNumbers title="Terminal"
npx nocta-ui add avatar
```
</Step>
<Step>
Then import the component:

```tsx lineNumbers title="Import"

```
</Step>
</Steps>

## Fallback Handling

The Avatar component gracefully handles image loading failures with multiple fallback options.

    // Invalid image with initials fallback
    <Avatar src="invalid-image.jpg" fallback="JD" alt="John Doe" />

    // Initials from name (auto-generated)
    <Avatar fallback="Sarah Connor" />

    <Avatar fallback="AB" />

## API Reference

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `src` | `string` | - | The image source URL |
| `alt` | `string` | `''` | Alt text for the image |
| `fallback` | `string` | - | Text to generate initials from, or custom initials |
| `size` | `'xs' \| 'sm' \| 'md' \| 'lg' \| 'xl' \| '2xl'` | `'md'` | The size of the avatar |
| `variant` | `'circle' \| 'square'` | `'circle'` | The shape variant |
| `status` | `'online' \| 'offline' \| 'away' \| 'busy' \| null` | `null` | Status indicator |
| `className` | `string` | `''` | Additional CSS classes to apply |

The Avatar component also accepts all standard HTML div element props through the `...props` spread operator.
